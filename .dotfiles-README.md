# Dotfiles (bare repo)

Managed with a bare git repo — files tracked in-place at their native paths.

## Usage

```bash
alias dotfiles='git --git-dir=$HOME/.dotfiles --work-tree=$HOME'

dotfiles status
dotfiles add ~/.some-config
dotfiles commit -m "update something"
dotfiles push
```

## Setup on a new machine

```bash
# 1. Clone bare
git clone --bare https://github.com/NetanelK/dotfiles.git $HOME/.dotfiles

# 2. Define alias
alias dotfiles='git --git-dir=$HOME/.dotfiles --work-tree=$HOME'

# 3. Configure
dotfiles config --local status.showUntrackedFiles no
dotfiles config --local core.excludesFile $HOME/.dotfiles-gitignore

# 4. Checkout (backup conflicts first)
dotfiles checkout 2>&1 | grep -E "^\s+" | awk '{print $1}' | \
  xargs -I{} mkdir -p backup && xargs -I{} mv {} backup/{}
dotfiles checkout

# 5. Set up Claude settings from template
cp ~/.claude/settings.json.template ~/.claude/settings.json
# Then edit ~/.claude/settings.json and fill in YOUR_AUTH_TOKEN_HERE
```

## Secrets (not tracked)

- SSH private keys (`~/.ssh/id_ed25519`, `~/.ssh/*.pem`)
- `~/.claude/settings.json` (has auth token — use template)
- `~/.claude/zscaler-root-ca.pem`
- AWS credentials (`~/.aws/credentials`)

## After cloning

1. `brew bundle --file=<(dotfiles show main:Brewfile)` — or save Brewfile separately
2. Copy SSH keys from secure backup
3. `gh auth login`
4. `saml2aws login -a ovp-devops`
